# 2026-05-17 — Phase 2: TS v5.6 移植・完了

`packages/v5.6/` を追加し、TS 5.6 でも同じ convert-dts パイプラインを通すよう scripts-common を flexibility 化した作業の記録。`pnpm run check-all` exit 0 で完走。

## v5.6 パッケージ作成

- `packages/v5.6/` を `cp -r packages/v5.7 packages/v5.6` で作成し、`version-config.json` / `source/scripts/typescript-version.txt` (= `5.6.3`) / 各 `package.json` の name / description / typescript dep を v5.6 系へ書き換え。
- `packages/v5.6/source/scripts/package-root.mts` のコメントも v5.6 に更新。
- `pnpm install --no-frozen-lockfile` でワークスペースに `typescript@5.6.3` を追加取り込み。

## scripts-common の version-flexibility 化

TS 5.7 で typed-array (`Int8Array` 等) が generic 化された (`Int8Array<TArrayBuffer extends ArrayBufferLike>`) ことが convert-dts の string-match パターンに広く影響していたため、両 form を受理する形に書き換えた。

### 共通ヘルパー (`convert-dts/common.mts`)

- `typedArrayInterfaceStartRegexp(typeName)` — `interface Int8Array {` と `interface Int8Array<TArrayBuffer extends ArrayBufferLike> {` (および `= ArrayBufferLike` 付きの default 形) の両方にマッチ。
- `typedArrayRefRegexSource(typeName)` — `Int8Array(?:<ArrayBufferLike>)?` の regex 断片。
- `typedArrayUnionRegexSource(typeNames)` — 上の `|` 連結。
- `typedArrayThisRegexSource(typeName)` — `(?:this|${typeName})` — TS 5.7 が `array: this` を使うのに対し、TS 5.6 では `array: Int8Array` 等の具体名を使う差分に対応。
- `typedArrayRef(typeName, tsLibShape, genericArg?)` — 出力側。`hasTypedArrayGeneric` が true なら `Int8Array<ArrayBufferLike>`、false なら `Int8Array`。

### 型・関数の追加

- `TsLibShape = Readonly<{ hasTypedArrayGeneric: boolean }>`
- `tsLibShapeFor(tsVersion)` — TS バージョンから shape を導出 (TS ≥ 5.7 で `hasTypedArrayGeneric: true`)。
- `ConverterOptions` に `tsLibShape: TsLibShape` フィールド追加。

### convert / gen-lib-files の signature

- `convert(filename, config)` → `convert(filename, config, tsVersion)` に変更。
- `gen-lib-files.mts` で `getTypeScriptVersion()` を一度だけ呼び、各 file の convert に渡す。

## 次フェーズに繰り越す課題

1. **更に古いバージョン (5.5 ↓ → 4.5)** で追加の divergence が出る見込み:
   - TS 5.5 系: pipe operator 関連、Set methods (union/intersection) など upstream lib 追加点。
   - TS 5.4: `Object.groupBy`, `Promise.withResolvers` 等の lib 追加点。
   - 各バージョンで convert-dts に新たに `onNotFound: 'off'` / regex 化が必要なケースがある可能性大。
2. **AST ベース変換の検討**: regex-per-pattern は scale が悪い。Phase 2 中盤以降、`ts-morph` などの AST ベースに切り替えるか検討する余地あり。
3. **再生成決定性チェック**: `git diff --exit-code` を check-all に追加し、CI で「同じ TS バージョンに対して同じ output が出る」ことを担保。
4. **branded 側の diff** (`output-branded/diff/`) 生成は引き続き未対応。
5. **TypeScript 5.7 のみ存在する transformations** が v5.6 では no-op になっている可能性あり。差分の目視確認・テストで確認したい。
