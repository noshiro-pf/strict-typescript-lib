# 2026-05-17 — Phase 1: TS v5.7 移植・モノレポ化

webauto-typescript-shared をベースに strict-typescript-lib repo へ移植し、TS 5.7 向けの strict lib パッケージ生成パイプラインを動作させた作業の記録。

## 着手前の状態

- `main`: typescript-template scaffold (`src/`, `samples/`, `configs/rollup.config.ts` 等) 。
- `convert-to-monorepo` branch: `packages/package-a` `packages/package-b` のプレースホルダのみ持つモノレポテンプレが lay down 済み。docs/design.md は untracked で carry-over。

着手はこのブランチ上に対して追加コミットする想定で進めた。

## 主要な移植時の変更点

### 構成変更

- `convert-to-monorepo` の `packages/package-{a,b}` を削除し、以下を新設:
  - `packages/scripts-common/` — webauto をベースに移植した共通生成スクリプト (private workspace package)
  - `packages/v5.7/source/` — 生成エントリ (private workspace package)
  - `packages/v5.7/output/` — 非 branded 生成物 (private workspace package、commit 対象)
  - `packages/v5.7/output-branded/` — branded 生成物 (private workspace package、commit 対象)
- ルート設定 (`package.json`, `tsconfig.json`, `.prettierrc`, `.cspell.config.yaml`, `.gitignore`) を strict-ts-lib 向けに整備し、不要な scaffold (`configs/vite-config.mts`, `configs/tsconfig/tsconfig.vite.json`) を撤去。
- `pnpm-workspace.yaml` で workspace 対象を `packages/scripts-common` と `packages/v*/{source,output,output-branded}` に限定。生成された sub-packages (`packages/v*/output*/packages/**`) は workspace から除外。

### scripts-common (webauto 由来) の改修

- `STRICT_TS_LIB_PACKAGE_ROOT` env var + `packages/v*/version-config.json` で TS バージョン別の設定 (`libName` / `repo` / `license` / `typescriptVersionRange`) を注入する仕組みに変更。各 version の `source/scripts/package-root.mts` が env var をセットしてから scripts-common の command を dynamic import する。
- `@tier4/node-utils` 依存を `packages/scripts-common/src/functions/utils/{replace-with-no-match-check,replace-with-no-match-check-between-regexp,compose-mono-type-fns,generate-key-value-record-from-keys,slice-by-match}.mts` に内製化。`./utils/node-utils.mjs` の barrel で再 export。
- `convert-dts/common.mts` の `configs` 配列に branded エントリ (`useBrandedNumber: true, returnType: 'readonly'`) を追加し、mono の 2-config 構成を復活。
- ts-type-forge グローバルに存在しない `YearEnum` を、生成される `declare namespace NumberType { ... }` 内で `NumberType.YearEnum = SafeUint` として補完 (mono の `@noshiro/ts-type-utils` 互換)。
- `fetch-lib-files.mts` を `gh api` + `wget` から組み込み `fetch()` ベースに変更:
  - GitHub Contents API への非認証アクセス (auth が無くてもパブリックリポジトリは取得可能)
  - リクエストはシーケンシャル + リトライ付き (raw.githubusercontent.com への並列接続で connect timeout が出たため)
- `genLibFiles` で `output/lib-files/` と `output-branded/lib-files/` の両方をクリーンアップしてから書き込むように修正。
- gen-steps から不要コマンド (`publish-packages`, `gen-ci`, `gen-min`, `fmt-diff`, `get-package-dir-list`, `open-diff-in-code`) を削除。`gen-steps.mts` に branded 側の format step (`format output-branded/{lib-files,packages}`) を追加。
- `gen-packages.mts` を簡素化: `updateRootPackageJson` と `createLib` を削除、`tsVersion.startsWith(typescriptVersion)` の sanity check を除去、`publishConfig` の `npm.pkg.github.com` 指定を撤去 (npmjs.com デフォルトへ)、`peerDependencies.typescript` を `version-config.json` の `typescriptVersionRange` から注入。
- `gen-eslint-fixed.mts` は webauto と同形式を維持しつつ各 version 側に `source/configs/eslint.config.gen.mts` と `zz:eslint` script を新設して紐付け。
- 共有 `.prettierrc` に `packages/v*/source/temp/copied/**/*.d.ts` → `printWidth: 320` の override を追加 (webauto の `lib.decorators.d.ts` の `Class extends abstract new (...args: unknown) => ...` への regex マッチが prettier の改行で壊れる問題への対策)。

### check-all の更新

`scripts/cmd/check-all.mts` を strict-ts-lib のパイプライン用に再構成:

1. `pnpm i`
2. `pnpm run cspell`
3. `pnpm run md`
4. `pnpm run check:root` (root の eslint + `tsc -p tsconfig.json`)
5. **`pnpm run ws:gen`** (= `pnpm -r --filter './packages/v*/source' run gen:full`)
6. `pnpm run ws:test` (生成物の `tsc -p output*/tsconfig.json`)
7. `pnpm run ws:lint:fix`
8. `pnpm run fmt:full`

旧 scaffold の `build` / `doc` / `ws:test:cov` / `codemod:full` / `gh:backup-all` ステップは削除。

巨大な gen 出力に対応するため、`runCmdStep` を `ts-repo-utils` の `$` (内部で `child_process.exec` を使い 1MB の `maxBuffer` でバッファ) から `child_process.spawn({ shell: true, stdio: 'inherit' })` のストリーミング実行に置き換え。

### eslint 設定の調整

- `packages/scripts-common/eslint.config.mts`: 内製ユーティリティと変換ロジックに対して、`prefer-readonly-parameter-types` / `security/detect-non-literal-fs-filename` / `import-x/first` / `functional/immutable-data` 等を `off` に。生成ツール用の許容ルールとして整理。
- `.cspell.config.yaml`: 生成物 (`packages/v*/output*/{lib-files,packages,diff}`) と `source/temp/**` を `ignorePaths` に追加し、`merchantablity` 等のドメイン語を `words` に追加。
- `.gitignore` に `packages/v*/source/temp/` を追加。

### その他

- mono 由来の `clear-dir.mts` (`zx/globals` 依存) は未使用だったため削除。
- `packages/v5.7/output*/tsconfig.json` の `noLib: true` と共存させるため `lib: null` を採用 (空配列だと `TS5053: Option 'lib' cannot be specified with option 'noLib'` で衝突)。
- `packages/v5.7/source/scripts/package-root.mts` で `import.meta.dirname` を使い `fileURLToPath` 経由を回避 (Node 22+ で標準サポート、ESLint の `no-unused-vars` 警告を防ぐ)。

## 結果

- `pnpm run check-all` が exit 0 で完走。
- v5.7 で 17 個 × 2 = **34 個の sub-package** を生成:
  - 非 branded 例: `strict-ts-lib-v5.7-es5`, `strict-ts-lib-v5.7-dom-iterable`, ...
  - branded 例: `strict-ts-lib-v5.7-branded-es5`, `strict-ts-lib-v5.7-branded-dom-iterable`, ...
- `output/` と `output-branded/` の双方が `tsc -p tsconfig.json` でクリーンに通過。
